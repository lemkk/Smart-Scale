# Smart-Scale

# Smart Scale BLE 固件

基于 nRF52840 的蓝牙智能体重秤固件，通过 BLE Nordic UART Service（NUS）将重量和电池数据实时推送到手机。

------

## 硬件平台

| 项目       | 说明                                    |
| ---------- | --------------------------------------- |
| 主控芯片   | Nordic nRF52840（自定义 PCB）           |
| SDK        | nRF5 SDK + SoftDevice S140              |
| 称重传感器 | HX711（24位ADC，通道A，增益128）        |
| 电池监测   | SAADC AIN7（P0.31），P-ch MOS 分压控制  |
| 设备地址   | 8位拨码开关（DIP Switch），范围 000~255 |

------

## 功能概述

- **待机状态**：上电后系统进入低功耗待机，不做任何工作
- **开机**：按下按键后松开，系统启动，开始 BLE 广播，同时启动重量采样
- **关机**：长按按键 3 秒，断开 BLE 连接，进入 system-off 深度休眠
- **BLE 广播名称**：由 8 位拨码开关动态生成，格式为 `Smart Scale-XXX`（000~255）
- **数据推送**：连接后每秒发送一次数据，格式：`x.xxx kg BAT:xx%`
- **去皮（Tare）**：开机后短按按键，以当前重量为新零点，并保存到 Flash
- **低电量保护**：电量 ≤ 20% 时红灯闪烁；电量耗尽（≤3500mV）时自动关机

## GPIO 引脚定义

| 功能               | 引脚         | 说明                 |
| ------------------ | ------------ | -------------------- |
| HX711 DOUT         | P0.04        | 数据输入             |
| HX711 SCK          | P0.06        | 时钟输出             |
| 电源自锁           | P0.15        | 上电后拉高，保持供电 |
| 按键               | P0.17        | 按下低电平，内部上拉 |
| LED 蓝（状态指示） | P0.08        | 低电平亮             |
| LED 红（低电量）   | P0.12        | 低电平亮             |
| LED 绿（预留）     | P1.09        | —                    |
| 电池分压控制       | P0.29        | 高电平导通 P-ch MOS  |
| 电池ADC            | P0.31 / AIN7 | SAADC 通道0          |
| 拨码开关 bit0~bit7 | P0.20~P0.09  | 详见下方说明         |

### 拨码开关引脚映射

| 物理位（左→右） | bit  | 值   | GPIO                           |
| --------------- | ---- | ---- | ------------------------------ |
| 第1位           | bit0 | 1    | P0.20                          |
| 第2位           | bit1 | 2    | P0.22                          |
| 第3位           | bit2 | 4    | P0.24                          |
| 第4位           | bit3 | 8    | P1.00                          |
| 第5位           | bit4 | 16   | P1.02                          |
| 第6位           | bit5 | 32   | P1.04                          |
| 第7位           | bit6 | 64   | P1.06                          |
| 第8位           | bit7 | 128  | P0.09（NFC引脚，需切换为GPIO） |

> **注意**：P0.09 为 NFC 复用引脚。固件在首次运行时会自动写入 UICR 寄存器并复位，将其切换为普通 GPIO 模式。此操作仅执行一次。

| -------------- | ---------- | ---------- |
| 待机           | 熄灭       | 熄灭       |
| BLE 广播中     | 500ms 闪烁 | 熄灭       |

## 按键操作

| 操作                 | 效果                           |
| -------------------- | ------------------------------ |
| 短按（任意时长松开） | 待机→开机；开机中→去皮（Tare） |
| 长按 3 秒            | 开机→关机（进入深度休眠）      |

------

## 数据格式

每秒通过 BLE NUS 发送一条字符串：

```
x.xxx kg BAT:xx%\r\n
```

示例：

```
1.234 kg BAT:85%
-0.012 kg BAT:72%
```

负值表示去皮后的残差偏差。

------

## 重量采样流程

每 1 秒触发一次状态机，全程无阻塞（每步回调 < 1ms）：

```
[1s主定时器] → PHASE0: 导通电池分压
     ↓ 50ms
PHASE1: ADC 采样（16次平均），关断分压
     ↓ 100ms
PHASE2: HX711 第1次读数
     ↓ 100ms
PHASE3: HX711 第2次读数
     ↓ 100ms
PHASE4: HX711 第3次读数 → 计算均值 → 发送BLE数据
```

电池电压采用 4 次滑动平均，3500mV~4200mV 对应 0%~100%。

| ---------- | ---- | ----------------------------------------------- |

## Flash 存储（FDS）

去皮基准值（`Weight_Maopi`）通过 Nordic FDS 持久化存储：

- **保存时机**：执行去皮操作后自动写入 Flash
- **恢复时机**：每次开机初始化时从 Flash 读取，无需重新去皮
- FDS File ID: `0x0001`，Record Key: `0x0001`

------

## Flash 分区规划

nRF52840 共 1MB Flash，分为 256 页（每页 4KB）。下表为完整分区规划，标注了当前实现状态。

```
区域                起始地址    结束地址    大小       页数    状态
─────────────────────────────────────────────────────────────────────
MBR                0x00000    0x01000    4KB        1页    ✅ 已有（SoftDevice内置）
SoftDevice S140    0x01000    0x27000    152KB      38页   ✅ 已有
App                0x27000    0x89000    392KB      98页   ✅ 当前固件
OTA 接收区          0x89000    0xEB000    392KB      98页   🔲 规划中
user param / FDS   0xEB000    0xED000    8KB        2页    🔲 规划中（当前FDS在App区内）
sys param          0xED000    0xEF000    8KB        2页    🔲 规划中
BLE peer param     0xEF000    0xF2000    12KB       3页    🔲 规划中
Bootloader         0xF2000    0xFE000    48KB       12页   🔲 规划中
OTA Setting Backup 0xFE000    0xFF000    4KB        1页    🔲 规划中
OTA Setting        0xFF000    0x100000   4KB        1页    🔲 规划中
─────────────────────────────────────────────────────────────────────
合计                                     1024KB     256页  ✓
```

### 当前链接器配置

当前工程（无 Bootloader）仅使用 App 区，链接器参数如下：

```
FLASH_PH_START = 0x00000    # Flash 物理起始
FLASH_PH_SIZE  = 0x100000   # Flash 物理总大小（1MB）
RAM_PH_START   = 0x20000000 # RAM 物理起始
RAM_PH_SIZE    = 0x40000    # RAM 物理总大小（256KB）

FLASH_START    = 0x27000    # 应用程序入口（SoftDevice之后）
FLASH_SIZE     = 0x62000    # 应用程序可用 Flash（392KB）
RAM_START      = 0x20002AE8 # 应用程序 RAM 起始（SoftDevice之后）
RAM_SIZE       = 0x3D518    # 应用程序可用 RAM（~245KB）
```

> **注意**：工程文件 `.emProject` 中 `FLASH_SIZE` 当前仍为 `0xd9000`，需更新为 `0x62000` 以匹配最新分区规划，否则链接器会以旧值为准。

### 引入 Bootloader 后的变化

接入 OTA Bootloader 后，App 区链接参数将保持不变（`FLASH_START=0x27000`，`FLASH_SIZE=0x62000`），Bootloader 独立编译并烧录到 `0xF2000`，两者通过 MBR + SoftDevice 的标准跳转机制协作。

### Flash 空间使用参考

```
0x00000000 ┌─────────────────────────┐
           │   SoftDevice S140       │  156 KB
0x00027000 ├─────────────────────────┤  ← FLASH_START（应用入口）
           │   应用程序代码            │
           │   FDS持久化存储           │  共 392 KB
0x00089000 └─────────────────────────┘  ← FLASH_START + FLASH_SIZE
```

## 编译与烧录

1. 使用 SEGGER Embedded Studio 打开 `ble_app_uart_pca10056_s140.emProject`
2. 确认 nRF5 SDK 路径配置正确
3. 编译并通过 J-Link 烧录固件
4. 首次烧录后，固件会自动配置 UICR 并触发一次复位（正常现象）
